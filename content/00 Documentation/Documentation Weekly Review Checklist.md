# Weekly Review Checklist

## Purpose

Keep your vault healthy. Prevent chaos. Ensure your research is moving forward.

## Before You Start

- [ ] Open `00 Dashboard`
- [ ] Close all other notes
- [ ] Set a timer for 30 minutes

## The Checklist

### 1. Dashboard Health (5 min)

- [ ] Vault Overview shows all expected notes
- [ ] No broken Dataview queries
- [ ] Recently Modified shows this week's work
- [ ] Untagged Notes list is manageable (under 10)

### 2. Orphaned Notes (10 min)

- [ ] Open `03 Permanent Notes/Index`
- [ ] Find Permanent Notes with no "Linked By" (no incoming links)
- [ ] For each orphaned note:
  - [ ] Does it need links TO other notes? Add them.
  - [ ] Does it need links FROM other notes? Find where it belongs.
  - [ ] If it's not useful, move to `99 Archive/`

### 3. Literature Note Quality (5 min)

- [ ] Open `02 Literature Notes/Index`
- [ ] Check for notes with empty "Critical Analysis" section
- [ ] Add critique to any that are just summaries

### 4. Source Backlog (5 min)

- [ ] Open `01 Sources`
- [ ] Count unread sources
- [ ] If more than 5, schedule reading time
- [ ] If any sources are stale (no Literature Note after 2 weeks), prioritize or discard

### 5. Project Status (5 min)

- [ ] Open `04 Projects/Index`
- [ ] Check status of each active project
- [ ] Update any "draft" that should be "working"
- [ ] Move any "stable" project to `05 Outputs/`

### 6. Tags and Metadata (5 min)

- [ ] Run through Untagged Notes list
- [ ] Add relevant tags
- [ ] Check for inconsistent tags (e.g., `Policy` vs `policy`)
- [ ] Standardize to lowercase, hyphenated format

### 7. AI Assistance (5 min)

- [ ] Pick one note that feels "stuck"
- [ ] Run Copilot "Find Connections"
- [ ] Add any genuinely useful links
- [ ] Ignore suggestions that don't fit

### 8. Archive Cleanup (5 min)

- [ ] Open `99 Archive`
- [ ] Anything that can be deleted? (Be ruthless)
- [ ] Anything that should be un-archived?

### After Review

- [ ] Close all notes
- [ ] Open `00 Dashboard`
- [ ] Note your current project count, note count, and any concerns
- [ ] Schedule next week's review

## Monthly Deep Review (1 hour, first Sunday of month)

- [ ] Re-read your oldest Permanent Notes
- [ ] Have your views changed? Update them.
- [ ] Check concept maturity: any Candidates ready to become Supported?
- [ ] Review Outputs: anything ready to publish?
- [ ] Check Research Standards: are you following your own rules?
- [ ] Update this checklist if needed

## Red Flags (Act Immediately)

| Symptom | Problem | Fix |
|---------|---------|-----|
| Dashboard shows errors | Broken Dataview query | Check note in Source mode |
| Templater not filling dates | Corrupted template | Rebuild template in Source mode |
| Copilot not responding | API issue | Check settings and key |
| More than 20 untagged notes | Falling behind on organization | Schedule extra tagging session |
| Orphaned notes increasing | Not linking ideas | Spend next session only on connections |
| No notes created this week | Research stall | Re-read motivation, pick one small source |

## Success Metrics

- [ ] All notes have complete metadata
- [ ] All Permanent Notes have at least one link
- [ ] All Literature Notes have critical analysis
- [ ] All Sources have corresponding Literature Notes
- [ ] Dashboard loads without errors
- [ ] You can find any note in under 10 seconds
